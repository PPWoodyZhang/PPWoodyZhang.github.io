promise浅理解

准备工作：

什么是回调函数？

官方解释：一个被当做参数的函数，即为回调函数。

我的理解就是，我定义的函数但是我没调用，却执行了

两种回调函数的种类

1).同步回调:
理解:立即执行，完全执行完了才结束，不会放入回调队列中
例子:数组遍历相关的回调函数/ Promise的excutor函数
2). 异步回调:
理解:不会立即执行，会放入回调队列中将来执行
例子:定时器回调/ ajax回调/ Promise的成功|失败的回调  

promise

是什么？

理解

promise是一步编程中新的解决方案，旧的解决方案是纯回调函数

具体表达：

(1)从语法上来说: Promise是一个构造函数 
(2)从功能上来说: promise对象用来封装一个异步操作并可以获取其结果 

状态

  • 由pedding改为resolved
  • 由pedding改为rejected

怎样使用？

为什么要使用promise？

1.指定回调函数的方式更加灵活:
旧的:必须在启动异步任务前指定
promise:启动异步任务=>返回promie对象=>给promise对象绑定回调函数
2.支持链式调用可以解决回调地狱问题
什么是回调地狱?回调函数嵌套调用，外部回调函数异步执行的结果是嵌套的回调函数执行的条件
回调地狱的缺点?不便 于阅读/不便于异常处理
解决方案? promise链式调用
终极解决方案? async/ await

递归思想（实例:二叉树深度）

什么是递归思想？

一种通过重复将问题分解为同类的子问题而解决问题的方法，直接或者间接调用自身函数或者方法的算法。简单地说，就是如果在函数中存在着调用函数本身的情况，这种现象就叫递归。

实质：递归中的”递”就是入栈，”归”就是出栈。

“递”的意思是将问题拆解成子问题来解决， 子问题再拆解成子子问题，…，直到被拆解的子问题无需再拆分成更细的子问题（即可以求解），

”归“是说最小的子问题解决了，那么它的上一层子问题也就解决了，上一层的子问题解决了，上上层子问题自然也就解决了

递归算法的一般形式：(java)

经典例题：斐波那契数列，猴子吃桃子 ，等。

这里我写下我刚做的一道简单的递归算法：二叉树的深度

输入一棵二叉树的根节点，求该树的深度。从根节点到叶节点依次经过的节点（含根、叶节点）形成树的一条路径，最长路径的长度为树的深度。

又题意可知：二叉树的深度 等于 左子树的深度 与右子树的深度 中的 最大值 +1。

 终止条件：当 root​ 为空，说明已越过叶节点，返回 深度 0

动态规划——数组中连续子数组的最大和（类型）

输入一个整型数组，数组中的一个或连续多个整数组成一个子数组。求所有子数组的和的最大值。

输入: nums = [-2,1,-3,4,-1,2,1,-5,4]
输出: 6
解释: 连续子数组 [4,-1,2,1] 的和最大，为 6。

了解了动态规划的思想：在于拆分子问题，记住过往，减少重复计算。

思路：

  • 穷举分析
  • 确定边界
  • 找出规律，确定最优子结构
  • 写出状态转移方程

由此可知确定转移方程：

dp[i] 代表以元素 nums[i]nums[i] 为结尾的连续子数组最大和。

当 dp[i - 1] >0 时：执行 dp[i] = dp[i-1] + nums[i]；
当 dp[i - 1]≤0 时：执行 dp[i] = nums[i]

用动态规划也可以解之前做过的青蛙跳阶的问题

一只青蛙一次可以跳上1级台阶，也可以跳上2级台阶。求该青蛙跳上一个 10 级的台阶总共有多少种跳法。

分析可知

边界为：当台阶数是1的时候或者2的时候，可以明确知道青蛙跳法。所以是f(1),f(2)

最优子结构：n>=3时，已经呈现出规律 f(n) = f(n-1) + f(n-2)

转移方程为：

f(n)={

1      n=1;

2      n=2;

f(n-1)+f(n-2)       n>=3

}

力扣剑指算法练习体验总结

        这个暑假开始认真的去练算法，学前端也要会啊，老为难人了；先从简单的来，每天一道，暑假差不多页做了有四十道了，不过我们用java写算法，暂时还没用javaScript。最开始学的java基础早就忘得差不多了，现在的状态还是，脑子里有思路，不知道应该用什么方法去实现，一点点查找方法，最后才把一道简单的算法题给做出来，真不容易，无奈笑…

        做了这四十几道，常用的思路就是指针，递归，再去利用栈，链表。刚刚才做过一道，利用递归的方法或者辅助栈的方式，生成二叉树的镜像。现在就简单总结一下吧。

题目是这样的：

请完成一个函数，输入一个二叉树，该函数输出它的镜像。

例如输入：

     4
   /   \
  2     7
 / \   / \
1   3 6   9
镜像输出：

     4
   /   \
  7     2
 / \   / \
9   6 3   1

思路：

(1)递归

        终止条件：root为空

        进行：定义变量temp去暂存root的左节点

                   运行：递归右节点，将返回值作为左节点

                              递归左节点，将返回值作为右节点

(2)辅助栈（队列）

        终止条件：root为空

        进行：当前节点弹出，让它的子节点进栈，同时交换子节点，返回根节点

let const var的区别以及rest参数

复习es6，遇到let，想起来这个问题，脑子里还是不太清晰。那就再复习总结一遍吧。

  • var

  1. 用var声明的变量既是全局变量，也是顶层变量

    注意：顶层对象，在浏览器环境指的是window对象，在 Node 指的是global对象

  2. 存在变量提升的情况

  3. 能够对一个变量进行多次声明，后面的会覆盖前面的

  4. 在函数中使用使用var声明变量时候，该变量是局部的（而如果在函数内不使用var，该变量是全局的）

  • let
  1. 只在let命令所在的代码块内有效
  2. 不存在变量提升
  3. 不允许在相同作用域中重复声明
  • const
  1. 声明一个只读的常量，一旦声明，常量的值就不能改
  2. const一旦声明变量，就必须立即初始化，不能留到以后赋值
  3. 实质是变量指向的那个内存地址所保存的数据不得改动

区别：

  • 变量提升（var存在，let、const不存在）
  • 暂时性死区（var不存在，let、const存在）
  • 块级作用域（var不存在，let、const存在）
  • 重复声明（var允许，let、const在同一作用域不允许）
  • 修改声明的变量（var、let可以，const不可以）

rest参数

  • (形式为’…变量名’)，用于获取函数的多余实参用来代替arguments。
  • rest参数只能作为函数的最后一个参数。
  • 真正意义上的数组，可以使用数组自带的API。

package-lock.json的作用以及promise

原来：npm5以前，是不会有package-lock.json这个文件，

现在：npm5以后，当你安装包的时候，npm都会生成或者更新package-lock.json（其中包node_modules中所有包的信息）

npm5以后的版本安装都不要加--save参数，它会自动保存依赖信息

作用：

  1. node_modules 删除后，可以使用npm install重新下载回来，这时package-lock.json就可以锁定版本号，防止自动升级
  • 重新npm install的时候速度就可以提升

问题：文件的读取无法判断执行顺序（文件的执行顺序是依据文件的大小来决定的）(异步api无法保证文件的执行顺序)

解决方案：回调嵌套==>promise(提高了代码的可读性)

  • Promise本身不是异步的，但往往都是内部封装一个异步任务

认识到服务端与客户端的不同。

服务端，是为客户端服务的。服务的内容诸如向客户端提供资源，保存客户端数据。是实现游戏特色化的重要途径，也是最直接可以通过游戏表现出来的技术。

客户端，或称为用户端，是指与服务器相对应，为客户提供本地服务的程序。除了一些只在本地运行的应用程序之外，一般安装在普通的客户机上，需要与服务端互相配合运行。

一般来说，客户端就是我们使用的电脑（包括我们使用的浏览器IE,Firefox）；服务端就是存放网页与数据库数据的服务器。
